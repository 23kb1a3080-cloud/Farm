METHOD 1: AUTOMATED (Recommended - 15 minutes total)
--------------------------------------------------------------------------------
1. Setup MongoDB Atlas (5 min)
   → Create account at https://www.mongodb.com/cloud/atlas
   → Create free M0 cluster
   → Get connection string

2. Launch EC2 Instance (5 min)
   → Ubuntu 22.04 LTS
   → t2.micro (free tier)
   → Open ports: 22, 80, 443, 5000

3. Connect via SSH and run:
   
   ssh -i your-key.pem ubuntu@YOUR_PUBLIC_IP
   
   wget https://raw.githubusercontent.com/23kb1a3080-cloud/Farm/main/deploy.sh
   chmod +x deploy.sh
   ./deploy.sh

4. Seed Database:
   
   cd ~/Farm
   source venv/bin/activate
   python3 seed.py

5. Access website:
   
   http://YOUR_PUBLIC_IP

   ├─ Network access: 0.0.0.0/0 (Allow anywhere)
   └─ Connection string format:

   ├─ SSH (22) - Your IP
   ├─ HTTP (80) - 0.0.0.0/0
   ├─ HTTPS (443) - 0.0.0.0/0
   └─ Custom TCP (5000) - 0.0.0.0/0

3. EC2 INSTANCE SPECS:
   ├─ AMI: Ubuntu Server 22.04 LTS
   ├─ Instance Type: t2.micro (1 vCPU, 1GB RAM)
   ├─ Storage: 20GB gp2
   └─ Region: Any (ap-south-1 recommended for India)

================================================================================
                        💡 USEFUL COMMANDS (After Deployment)
================================================================================

VIEW LOGS:
  sudo journalctl -u farm -f                  # Application logs (real-time)
  sudo tail -f /var/log/nginx/error.log       # Nginx errors

RESTART SERVICES:
  sudo systemctl restart farm                 # Restart Flask app
  sudo systemctl restart nginx                # Restart web server
  sudo systemctl restart farm nginx           # Restart both

CHECK STATUS:
  sudo systemctl status farm                  # Check app status
  sudo systemctl status nginx                 # Check nginx status

UPDATE APPLICATION:
  cd ~/Farm                                   # Navigate to app
  git pull origin main                        # Pull latest code
  sudo systemctl restart farm                 # Restart app

SEED DATABASE:
  cd ~/Farm                                   # Navigate to app
  source venv/bin/activate                    # Activate virtual env
  python3 seed.py                             # Run seeder

VIEW CONFIGURATION:
  nano ~/Farm/config.py                       # Edit config
  sudo nano /etc/nginx/sites-available/farm   # Edit nginx config
  sudo nano /etc/systemd/system/farm.service  # Edit systemd service

================================================================================
                        🐛 COMMON ISSUES & SOLUTIONS
================================================================================

ISSUE: Can't access website at http://PUBLIC_IP
SOLUTION:
  1. Check security group allows port 80
  2. Verify services: sudo systemctl status farm nginx
  3. Check logs: sudo journalctl -u farm -n 50

ISSUE: 502 Bad Gateway Error
SOLUTION:
  1. Restart services: sudo systemctl restart farm nginx
  2. Check logs: sudo journalctl -u farm -n 50
  3. Verify Gunicorn is running: ps aux | grep gunicorn

ISSUE: Database Connection Error
SOLUTION:
  1. Check config.py has correct MongoDB URI
  2. Verify MongoDB Atlas Network Access allows 0.0.0.0/0
  3. Test connection: cd ~/Farm && source venv/bin/activate
     python3 -c "from database import db; print(db.name)"

ISSUE: Application not updating after git pull
SOLUTION:
  cd ~/Farm
  git pull origin main
  source venv/bin/activate
  pip install -r requirements.txt
  sudo systemctl restart farm

                        🔒 SECURITY RECOMMENDATIONS
================================================================================

IMMEDIATE ACTIONS:
  ✅ Change SSH security group from "Anywhere" to "My IP"
  ✅ Use strong MongoDB password
  ✅ Change SECRET_KEY in config.py
  ✅ Enable UFW firewall:
     sudo ufw allow 22,80,443,5000/tcp
     sudo ufw enable
